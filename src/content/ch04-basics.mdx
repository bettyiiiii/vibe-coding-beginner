## 变量与数据类型

### 变量：给东西起名字

**一句话理解：** 变量就是一个「贴了标签的盒子」，里面装着数据。

```javascript
const message = '你好，世界'
let score = 100
```

<AiExplainBtn explanation="`const` 和 `let` 都是「定义变量」的关键字：`const` 定义的变量不能再改（常量），`let` 定义的可以再改。`=` 是「赋值」，把右边的值装进左边的盒子。" />

**生活比喻：** `let score = 100` 就像在笔记本上写「score: 100」——后面可以随时把 100 改成别的数字。

### 常见数据类型

| 类型 | 例子 | 生活比喻 |
|------|------|---------|
| 字符串 String | `'你好'` | 一段文字 |
| 数字 Number | `42` | 一个数字 |
| 布尔 Boolean | `true` / `false` | 开关的开/关 |
| 数组 Array | `[1, 2, 3]` | 一列收纳盒 |
| 对象 Object | `{ name: '小明' }` | 一个有多个标签的收纳盒 |

<Callout type="tip" title="初学者不用背">
遇到不懂的数据类型，直接问 AI：「请用一个生活比喻解释 JavaScript 里的 [类型名]」。
</Callout>

## 函数：把动作打包

### 什么是函数？

**一句话理解：** 函数就是把一段代码「打包」，起个名字，需要的时候叫它出来执行。

```javascript
function greet(name) {
  return '你好，' + name + '！'
}

greet('小明')   // 输出：你好，小明！
```

<AiExplainBtn explanation="`function greet(name)` 定义了一个叫 `greet` 的函数，`name` 是「参数」（传入的值）。`return` 是函数的「返回值」——函数执行完后「给回」你的结果。" />

**生活比喻：** 函数就像微波炉——你放进去食物（传入参数），它加热完给你（return）热好的食物。

### 为什么要用函数？

1. **复用**：写一次，到处用
2. **清晰**：给一段代码起个名字，读代码更容易
3. **维护**：改一处，所有用的地方都跟着改

## 条件判断：让程序做决定

### If / Else

```javascript
const age = 20

if (age >= 18) {
  console.log('可以投票')
} else {
  console.log('还不可以投票')
}
```

<AiExplainBtn explanation="`if (条件)` 判断括号里的条件是不是「真」——是真就执行第一块代码，否则执行 `else` 后面的代码块。`>=` 是「大于等于」的意思。" />

**生活比喻：** 就像「如果（if）下雨，就带伞；否则（else）不带」。

### 常用比较符号

| 符号 | 意思 | 例子 |
|------|------|------|
| `===` | 等于 | `name === '小明'` |
| `!==` | 不等于 | `age !== 18` |
| `>` `>=` | 大于 / 大于等于 | `score > 90` |
| `<` `<=` | 小于 / 小于等于 | `count < 5` |

<Callout type="warning" title="一个 = 还是三个 ==">
- `=` 是「赋值」（把值装进盒子）
- `===` 是「比较」（看看两个东西相等吗）

这是初学者最容易混的地方！
</Callout>

## 用 AI 帮你练习

### 练习方法：让 AI 出题目

```
你是一个编程老师。

请出 3 道 JavaScript 基础题，主题是【变量 / 函数 / 条件判断】，
每题给出题目要求、示例代码框架（留空让我填），
做完后帮我检查答案并解释。
```

### 实战：让 AI 帮你写一个计算器

```
你是一个资深前端工程师。

请帮我写一个简易计算器网页，要求：
1. 能做加、减、乘、除
2. 用 HTML + CSS + JS 单文件
3. 代码加详细注释，我是初学者
4. 每一步都解释为什么这么写
```

<AiExplainBtn explanation="这个练习会用到所有基础知识：变量（存储数字）、函数（计算逻辑）、条件判断（处理不同运算符）。做完这个，你对基础语法的理解会深很多。" />
