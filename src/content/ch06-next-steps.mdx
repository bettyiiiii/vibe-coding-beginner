## 实战项目：做一个完整的「个人记账本」应用

现在，让我们把前面学的所有知识**串起来**，做一个真正能用的完整应用。

### 功能需求

1. **添加开销**：输入金额、类别、备注、日期
2. **查看列表**：显示所有开销，最新的在最上面
3. **数据统计**：显示今日/本周/本月总开销
4. **数据持久化**：用 localStorage 保存，刷新不丢失
5. **删除功能**：可以删除某条记录

---

### 第一步：拆解需求（提示词工程）

不要直接说「帮我做一个记账本」，而是拆解成小步骤：

```
你是一个资深前端工程师。

我想做一个「个人记账本」网页应用，请分步实现。

【第一步：先做静态页面】
请帮我写 HTML 结构：
- 顶部标题
- 输入区域（金额、类别下拉、备注、日期）
- 添加按钮
- 统计区域（今日/本周/本月总开销）
- 列表区域（显示所有记录）

【要求】
1. 纯 HTML + CSS + JS，单文件
2. 先写死数据（不用 JS 生成），让我看布局
3. 风格简洁，绿色主题
4. 代码加详细注释

请只输出第一 step 的代码，我确认 OK 后再继续。
```

---

### 第二步：让 AI 实现交互逻辑

```
谢谢，静态页面 OK 了。

现在请实现交互逻辑：
1. 点击「添加」按钮，把输入的数据加到列表里
2. 列表用 JS 动态生成（不用写死的 HTML）
3. 每次添加后，清空输入框
4. 更新统计数字（今日/本周/本月总开销）

请输出修改后的完整代码，并解释你改了哪些地方。
```

---

### 第三步：加数据持久化

```
很好，现在请加上数据持久化：
1. 每次添加/删除记录时，保存到 localStorage
2. 页面加载时，从 localStorage 读取历史数据
3. key 用 'expense-records'

请输出修改后的完整代码。
```

---

### 第四步：加删除功能

```
最后，请加上删除功能：
1. 每条记录右侧有一个「删除」按钮
2. 点击后从列表和 localStorage 里都删除
3. 加一个确认弹窗（confirm（）），防止误删

请输出最终完整代码。
```

---

### 保存和运行

和之前的例子一样：
1. 复制 AI 回复中的代码
2. 保存为 `expense-tracker.html`
3. 双击打开，开始用！

试着：
- ✅ 添加几笔开销
- ✅ 刷新页面，看数据是否还在
- ✅ 删除一条记录
- ✅ 看看统计数字是否正确

## 项目完成后，如何部署上线？

做完了项目，你一定想让朋友也能访问。**部署上线**就是把你的代码放到互联网上，任何人都能访问。

### 方法一：GitHub Pages（免费，适合静态网页）

```
第一步：注册 GitHub 账号（github.com）
第二步：新建一个仓库（Repository），名字叫 my-vibe-coding
第三步：把你的 .html 文件上传到仓库
第四步：在仓库设置里开启 GitHub Pages
第五步：你会得到一个网址，比如：
  https://yourname.github.io/my-vibe-coding/
```

### 方法二：Vercel（免费，更适合完整项目）

```
第一步：注册 Vercel 账号（vercel.com）
第二步：安装 Vercel CLI，或者直接拖拽文件夹上传
第三步：Vercel 会自动给你一个网址，比如：
  https://my-project.vercel.app/
```

> 💡 **推荐**：静态网页（单文件）用 **GitHub Pages**，有构建流程的项目用 **Vercel**。

## 你的第一个「独立项目」创意清单

### 进阶级（1 周完成）

5. **天气查询网页** —— 用户输入城市，调用天气 API 显示结果

## 写给初学者的几句话

### 1. 不要和「专业程序员」比

你学 Vibe Coding 的目的，不是为了去大厂当程序员。

**你的优势是**：快速把想法变成作品。

专业程序员花 3 天写的代码，你用 AI 3 小时搞定——虽然代码不够优雅，但**作品已经上线了**。

---

### 2. 卡住的时候怎么办？

**80% 的编程问题，AI 能直接帮你解决。**

剩下 20%，去社区问。问的时候记得附上：
- 你想做什么
- 你试了什么
- 具体的错误信息

---

### 3. 最重要的是：持续做出东西

不要陷入「学完再动手」的陷阱。

**边做边学，做的东西越多，你越厉害。**
