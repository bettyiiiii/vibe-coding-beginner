# Ch03 · 提示词工程：与 AI 高效沟通

> 提示词（Prompt）是你和 AI 之间的「翻译器」——你说得越清楚，AI 做得越好。

---

## 什么是提示词工程？

**提示词工程 = 如何说话让 AI 准确理解你的需求。**

| 糟糕的提示词 | 优秀的提示词 |
|----------|----------|
| "帮我做个网站" | "用 HTML+CSS 创建一个个人作品集网站，包含首页、关于我、联系我三个页面，使用响应式设计，主色调为深蓝色。" |
| "写个计算器" | "用 Python 写一个命令行计算器，支持加减乘除和平方运算，添加错误处理（如除零错误），包含中文注释。" |

> 💡 **核心原则**：AI 不知道你在想什么——你必须说清楚。

---

## 提示词四要素（万能模板）

任何高质量的提示词，都包含这四个要素：

```
【角色】你是谁？（让 AI 切换到正确的专家模式）
【背景】上下文是什么？（让 AI 理解前提条件）
【任务】具体要做什么？（清晰描述输出要求）
【约束】有什么限制？（格式、长度、风格、禁用内容）
```

### 实例对比

#### ❌ 糟糕示例

```
帮我写一个登录页面
```

#### ✅ 优秀示例（四要素齐全）

```
【角色】你是一位资深的前端开发工程师，精通 React 和 Tailwind CSS。

【背景】我正在开发一个在线教育平台，需要用户登录功能。

【任务】请帮我创建一个登录页面，包含：
1. 邮箱输入框
2. 密码输入框（带显示/隐藏切换）
3. 登录按钮
4. "忘记密码"链接
5. "还没有账号？注册"链接

【约束】
- 使用 React + Tailwind CSS
- 响应式设计（移动端适配）
- 添加表单验证（邮箱格式、密码长度）
- 代码包含详细注释
- 输出完整可运行的代码（单个文件）
```

## 三层拆解法（处理复杂需求）

当你的需求很复杂时，不要试图一次说完——**分步骤拆解**。

### 错误示范（一次性说太多）

```
帮我做一个完整的电商网站，包含用户注册、商品展示、购物车、订单管理、支付功能，用 React + Node.js + MongoDB，要安全、高性能、好看……
```

> 结果：AI 会混乱，生成的代码质量差。

### 正确示范（分步骤）

#### 第一步：让 AI 帮你规划

```
我想做一个电商网站，但我不太确定如何开始。
请帮我拆解成 5-8 个开发步骤，每个步骤清晰、可独立测试。
技术栈：React + Node.js + MongoDB
```

#### 第二步：让 AI 完成第一步

```
好的，我现在想完成第一步：【用户注册和登录功能】。

请帮我：
1. 设计数据库 User 表的字段
2. 写注册 API（/api/register）
3. 写登录 API（/api/login）
4. 用 JWT 实现身份验证

要求：代码包含详细注释，每步都可独立测试。
```

#### 第三步：测试，再继续

```
第一步的代码我已经运行测试通过了。
现在继续第二步：【商品展示页面】。
```

> 🏆 **核心技巧**：复杂项目 = 让 AI 先拆解 → 一步步完成 → 每步测试通过再继续。

## 进阶技巧：让 AI 解释代码

**不要只让 AI 写代码——让它解释代码，帮你学习。**

### 提示词模板

```
我有一段代码，请帮我逐行解释：

【代码】
（把 AI 生成的代码粘贴在这里）

请：
1. 逐行解释每行代码的作用
2. 如果有我不懂的概念，用生活化比喻解释
3. 指出可能的问题或改进点
```

### 实例

**你**：
```
请帮我用 Python 写一个函数，判断一个数是不是质数。
```

**AI 生成**：
```python
def is_prime(n):
    if n < 2:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False
    return True
```

**你**（重要！）：
```
谢谢你。请逐行解释这段代码：
1. 为什么 n < 2 就返回 False？
2. range(2, int(n**0.5) + 1) 是什么意思？
3. n % i == 0 是什么意思？
4. 用生活化比喻解释整个逻辑。
```

**AI 解释**（示例）：
```
1. 质数是大于 1 且只能被 1 和自身整除的数，所以小于 2 的不是质数。
2. range(2, int(n**0.5) + 1) 是试除法优化——只需要检查到 √n。
3. n % i 是取余运算，如果余数为 0，说明 n 能被 i 整除，不是质数。
4. 生活化比喻：判断一个人是不是"素数人"——他只能被 1 和他自己"整除"（交朋友）。
```

> 🎓 **学习建议**：每次 AI 生成代码后，**至少问一次"请解释"**——这是用 AI 学编程的最快方式。

## 常见提示词模式（可直接套用）

### 模式 1：代码生成

```
【角色】资深 [语言/框架] 开发工程师

【任务】请帮我创建 [功能描述]

【要求】
1. 使用 [技术栈]
2. 包含详细注释
3. 遵循 [代码规范，如 Airbnb / PEP8]
4. 输出完整可运行代码

【输出格式】先解释设计思路，再给出代码
```

---

### 模式 2：代码调试

```
【背景】我正在做一个 [项目描述]

【问题】运行代码时报错：
（粘贴完整错误信息）

【代码】
（粘贴相关代码）

【我已经尝试过】
1. （描述你试过的方法）

请帮我：
1. 找出问题原因
3. 给出修复后的完整代码
4. 解释为什么会出现这个问题，如何避免
```

---

### 模式 3：代码优化

```
【背景】这是我写的代码：
（粘贴代码）

【问题】这段代码 [运行太慢 / 不够优雅 / 有重复代码]

请帮我：
1. 指出代码中的问题（性能 / 可读性 / 最佳实践）
2. 给出优化后的代码（包含注释）
3. 解释优化思路和带来的好处
```

---

### 模式 4：概念解释

```
我正在学习 [主题，如：JavaScript 闭包 / Python 装饰器]。

请用以下方式解释：
1. 简单定义（一句话）
2. 生活化比喻
3. 代码示例（由浅入深，3 个示例）
4. 常见误区
5. 实际应用场景
```

## 实战演练：写一个待办清单（ToDo List）

### 第一步：需求描述

```
【角色】你是资深前端工程师，精通 React Hooks 和 Tailwind CSS。

【任务】帮我创建一个待办清单（ToDo List）网页应用。

【功能需求】
1. 显示待办列表（每个待办有文字和完成状态）
2. 添加新待办（输入框 + 添加按钮）
3. 标记完成（点击文字划线 + 灰色）
4. 删除待办（每个待办右侧有删除按钮）
5. 统计未完成数量（如"还有 3 项待完成"）

【技术要求】
1. 使用 React + Tailwind CSS
2. 数据用 useState 管理
3. 组件拆分清晰（TodoItem、TodoList、TodoInput）
4. 添加入场/退场动画（可选）

【输出要求】
1. 先解释组件设计思路
2. 给出完整可运行代码（单文件，方便测试）
3. 代码包含详细注释
```

### 第二步：让 AI 解释代码

> （AI 生成代码后，你的关键动作！）

```
谢谢你。请帮我解释这段代码：

1. useState 的作用是什么？为什么需要它？
2. 每个组件的职责是什么？为什么要拆分？
3. 添加新待办的逻辑是怎样的？（用流程图或步骤说明）
4. 标记完成功能是如何实现的？
5. 如果我想要"编辑待办"功能，应该改哪里？

请用初学者能理解的方式解释。
```

### 第三步：扩展功能

```
代码我已经理解并运行成功了！
现在我想添加以下功能：
1. 过滤显示（全部 / 未完成 / 已完成）
2. 清空已完成
3. 本地存储（刷新页面不丢失数据）

请帮我实现，并解释实现思路。
```

## 避免常见陷阱

### 陷阱 1：提示词太模糊

| ❌ 模糊 | ✅ 清晰 |
|----------|----------|
| "做个网站" | "用 HTML+CSS 做一个个人作品集网站，包含首页、作品展示、关于我、联系我 4 个页面" |
| "写个游戏" | "用 HTML5 Canvas 写一个贪吃蛇游戏，方向键控制，显示分数" |
| "帮我修复" | "我的 React 代码报错：Cannot read property 'map' of undefined，代码如下：[代码]" |

---

### 陷阱 2：盲目信任 AI 输出

**AI 会犯错！** 常见错误类型：

| 错误类型 | 例子 | 如何发现 |
|----------|--------|----------|
| 编造不存在的 API | `import NonExistentLibrary from 'fake-lib'` | 运行后报错"模块未找到" |
| 逻辑错误 | 循环条件写错，导致无限循环 | 程序卡死或输出异常 |
| 忽略边界情况 | 未处理空数组、null、undefined | 特定输入时报错 |
| 安全问题 | 直接使用用户输入拼 SQL（SQL 注入风险） | 学习基本安全知识，或明确提示"注意安全" |

> ⚠️ **铁律**：AI 生成的代码，**必须运行测试**，不要直接用于生产环境！

---

### 陷阱 3：提示词太长，AI "失忆"

**问题**：提示词超过 2000 字后，AI 可能会忘记前面的内容。

**解决方案**：
1. 拆分成多个对话（一个对话解决一个问题）
2. 使用"请记住以下要求"（让 AI 复述关键点）
3. 换用支持更长上下文的模型（如 Claude 3.5，支持 200K tokens）

### 练习 1：基础级

要求：
- 可以设置分钟和秒
- 开始/暂停/重置按钮
- 时间到 0 时播放提示音

### 练习 2：进阶级

写一个提示词，让 AI 帮你「优化以下代码的性能」：
```python
# 找出列表中出现次数最多的元素
def most_frequent(arr):
    max_count = 0
    max_item = None
    for i in range(len(arr)):
        count = 0
        for j in range(len(arr)):
            if arr[i] == arr[j]:
                count += 1
        if count > max_count:
            max_count = count
            max_item = arr[i]
    return max_item
```
