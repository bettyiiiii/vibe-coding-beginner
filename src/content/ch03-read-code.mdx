# 读懂 AI 写的代码

> 不用成为程序员也能读懂代码——你只需要知道「每一块在做什么」。

---

## 代码三件套：HTML / CSS / JS

网页代码通常由三种语言组成，各司其职：

| 语言 | 负责什么 | 生活比喻 |
|------|---------|---------|
| **HTML** | 内容和结构 | 房子的框架和墙壁 |
| **CSS** | 颜色和样式 | 装修和内饰 |
| **JS** | 交互和逻辑 | 房子的电路和智能设备 |

---

## 看一段真实的 AI 代码

下面是一段 AI 帮你写的 HTML 代码，我们逐行读懂它：

```html
<!DOCTYPE html>
<html lang="zh">
  <head>
    <meta charset="UTF-8" />
    <title>我的主页</title>
    <link rel="stylesheet" href="style.css" />
  </head>
  <body>
    <h1>你好，我是小明</h1>
    <p>我是一个初学者，正在学 Vibe Coding！</p>
  </body>
</html>
```

<AiExplainBtn explanation="这段代码的结构像一个「三明治」：最外层是 <html> 包裹一切；<head> 里放「设置」（字符编码、标题、样式文件）；<body> 里放「内容」（网页上真正显示的东西）。<h1> 是大标题，<p> 是段落。" />

## 向 AI 提问：让它解释代码

**这是最重要的技能：不用自己猜，让 AI 帮你解释。**

### 提示词模版

```
请帮我逐行解释下面这段代码，我是初学者，请用生活化比喻：

[把 AI 写的代码粘贴在这里]
```

### AI 会怎么回复？

AI 会给你：
1. **逐行解释**每一行在做什么
2. **生活化比喻**，帮你建立直觉
3. **哪些可以改**，改了会怎样

<Callout type="tip" title="小技巧">
把代码+「我是初学者，请用生活化比喻解释」一起发给 AI，效果最好。
</Callout>

## 常见代码模式：5 分钟认识它们

你不需要会写，但认识它们之后读代码会快很多：

### 变量（Variable）—— 给东西起名字

```javascript
const name = '小明'     // 把「小明」这个值，装进叫 name 的盒子
let age = 25            // let 和 const 都是「定义变量」的关键字
```

**生活比喻：** 变量就像贴了标签的收纳盒，`name` 是标签，`'小明'` 是里面的东西。

### 函数（Function）—— 把动作打包

```javascript
function sayHello(name) {
  return '你好，' + name + '!'
}
sayHello('小明')   // 输出：你好，小明！
```

**生活比喻：** 函数就像一个「小家电」——你按按钮（传入参数），它做固定的事情（函数体），然后给你结果（return）。

### 条件判断（If/Else）—— 让程序做决定

```javascript
if (age >= 18) {
  console.log('可以投票')
} else {
  console.log('还不可以投票')
}
```

**生活比喻：** 就像「如果下雨就带伞，否则不带」——程序根据条件选择不同的路。

### 循环（Loop）—— 重复做同一件事

```javascript
for (let i = 0; i < 3; i++) {
  console.log('这是第' + i + '次')
}
// 输出：这是第0次、这是第1次、这是第2次
```

**生活比喻：** 就像「把这个动作做 3 遍」——不用把代码抄 3 遍，用循环搞定。

<AiExplainBtn explanation="这些模式（变量、函数、条件、循环）是所有编程语言的「字母表」——不管是 JavaScript、Python 还是 C++，核心逻辑都是这几种。认识了它们，你就能读懂 80% 的代码。" />

## 调试技巧：当 AI 写的代码出错时

### 第一步：把错误信息直接给 AI

**不要试图自己猜错误！** 把浏览器控制台（F12 打开）里的红色报错信息**完整复制**给 AI，它会告诉你哪里错了、怎么修。

### 第二步：说清楚「我想做什么」

```
这段代码报错了，我的目标是【描述你想要的效果】，
以下是错误信息：

[粘贴完整错误信息]
```

### 第三步：让 AI 分步解释

如果 AI 修好的代码你还是看不懂，继续问：

```
谢谢，代码可以跑了。但我不太理解你改了哪里，
能逐行解释一下你的修改吗？
```

<Callout type="info" title="记住这个流程">
**代码出错 → 复制错误信息给 AI → AI 修复 → 让 AI 解释修改内容**

这就是 Vibe Coding 的完整调试流程，不需要你自己懂每一个错误。
</Callout>
