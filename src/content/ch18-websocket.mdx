# WebSocket 实时通信

> HTTP 是"问一句答一句"，WebSocket 是"保持通话"。

## 为什么需要 WebSocket？

### HTTP 的局限性

```
HTTP 通信：
客户端 → 服务器：给我新消息
服务器 → 客户端：没有新消息
（等待几秒）
客户端 → 服务器：给我新消息
服务器 → 客户端：没有新消息
（等待几秒）
客户端 → 服务器：给我新消息
服务器 → 客户端：有了！给你

问题：延迟高，浪费资源
```

### WebSocket 的优势

```
WebSocket 通信：
客户端 → 服务器：建立 WebSocket 连接
服务器 → 客户端：连接建立
（连接保持打开）
服务器 → 客户端：有新消息！
服务器 → 客户端：又有新消息！

优势：
- 实时推送
- 低延迟
- 节省带宽
- 双向通信
```

### 原生 WebSocket API

```javascript
// 客户端：连接 WebSocket 服务器
const ws = new WebSocket('ws://localhost:8080')

// 连接建立
ws.onopen = () => {
  console.log('WebSocket 连接已建立')
  
  // 发送消息
  ws.send(JSON.stringify({
    type: 'message',
    content: 'Hello WebSocket!'
  }))
}

// 接收消息
ws.onmessage = (event) => {
  const data = JSON.parse(event.data)
  console.log('收到消息:', data)
}

// 连接关闭
ws.onclose = () => {
  console.log('WebSocket 连接已关闭')
}

// 发生错误
ws.onerror = (error) => {
  console.error('WebSocket 错误:', error)
}

// 主动关闭连接
// ws.close()
```

## Socket.io 实战

### 为什么选择 Socket.io？

| 特性 | 原生 WebSocket | Socket.io |
|------|------------------|----------|
| 浏览器兼容 | 现代浏览器 | 所有浏览器（含 IE） |
| 断线重连 | ❌ | ✅ 自动重连 |
| 房间/命名空间 | ❌ | ✅ 内置支持 |
| 降级方案 | ❌ | ✅ 自动降级为轮询 |

### 安装 Socket.io

```bash
npm install socket.io socket.io-client
```

### 服务器端实现

```javascript
// server.js
const express = require('express')
const http = require('http')
const { Server } = require('socket.io')

const app = express()
const server = http.createServer(app)
const io = new Server(server, {
  cors: {
    origin: 'http://localhost:3000',
    methods: ['GET', 'POST']
  }
})

// 监听连接
io.on('connection', (socket) => {
  console.log('用户连接:', socket.id)
  
  // 监听客户端发送的消息
  socket.on('message', (data) => {
    console.log('收到消息:', data)
    
    // 广播给所有客户端（包括发送者）
    io.emit('message', {
      id: Date.now(),
      user: data.user,
      content: data.content,
      timestamp: new Date()
    })
    
    // 或只广播给其他客户端（不包括发送者）
    // socket.broadcast.emit('message', data)
  })
  
  // 监听断开连接
  socket.on('disconnect', () => {
    console.log('用户断开连接:', socket.id)
  })
})

server.listen(3001, () => {
  console.log('WebSocket 服务器运行在端口 3001')
})
```

## 实战：实时聊天室

### 服务器端

```javascript
// chat-server.js
const express = require('express')
const http = require('http')
const { Server } = require('socket.io')

const app = express()
const server = http.createServer(app)
const io = new Server(server, {
  cors: { origin: '*' }
})

// 存储在线用户
const onlineUsers = new Map()

io.on('connection', (socket) => {
  console.log('新连接:', socket.id)
  
  // 用户加入
  socket.on('join', (username) => {
    // 保存用户信息
    onlineUsers.set(socket.id, {
      id: socket.id,
      username: username,
      joinTime: new Date()
    })
    
    // 通知所有人
    io.emit('system', `${username} 加入了聊天室`)
    
    // 发送在线用户列表
    io.emit('online-users', Array.from(onlineUsers.values()))
  })
  
  // 接收聊天消息
  socket.on('chat-message', (data) => {
    const user = onlineUsers.get(socket.id)
    
    if (user) {
      // 广播消息
      io.emit('chat-message', {
        id: Date.now(),
        user: user.username,
        content: data.content,
        timestamp: new Date()
      })
    }
  })
  
  // 用户断开
  socket.on('disconnect', () => {
    const user = onlineUsers.get(socket.id)
    
    if (user) {
      // 通知所有人
      io.emit('system', `${user.username} 离开了聊天室`)
      
      // 移除用户
      onlineUsers.delete(socket.id)
      
      // 更新在线用户列表
      io.emit('online-users', Array.from(onlineUsers.values()))
    }
    
    console.log('用户断开:', socket.id)
  })
})

server.listen(3001)
```

      messages.appendChild(div)
      messages.scrollTop = messages.scrollHeight
    })
    
    // 系统消息
    socket.on('system', (msg) => {
      const messages = document.getElementById('messages')
      const div = document.createElement('div')
      div.className = 'message system'
      div.textContent = msg
      messages.appendChild(div)
    })
    
    // 更新在线用户列表

      userList.innerHTML = ''

## 房间（Rooms）

### 什么是房间？

房间是 Socket.io 的频道，用户加入房间后，只会收到该房间的消息。

```
例子：多人在线游戏
- 房间 A：游戏局 1
- 房间 B：游戏局 2
- 房间 C：游戏局 3

玩家 1 在房间 A，只能收到房间 A 的消息
```

### 使用房间

```javascript
// 服务器端
io.on('connection', (socket) => {
  // 加入房间
  socket.on('join-room', (roomId) => {
    socket.join(roomId)
    console.log(`用户 ${socket.id} 加入了房间 ${roomId}`)
    
    // 通知房间内其他用户
    socket.to(roomId).emit('user-joined', socket.id)
  })
  
  // 发送消息给房间
  socket.on('room-message', (data) => {
    const { roomId, content } = data
    
    // 发送给房间内所有人（包括自己）
    io.to(roomId).emit('room-message', {
      user: socket.id,
      content: content
    })
    
    // 或只发送给房间内其他人（不包括自己）
    // socket.to(roomId).emit('room-message', ...)
  })
  
  // 离开房间
  socket.on('leave-room', (roomId) => {
    socket.leave(roomId)
    console.log(`用户 ${socket.id} 离开了房间 ${roomId}`)
  })
})
```

## 命名空间（Namespaces）

### 什么是命名空间？

命名空间是 Socket.io 的顶层划分，用于隔离不同的应用或模块。

```
例子：一个应用有多个模块
- /chat：聊天模块
- /notifications：通知模块
- /game：游戏模块

每个命名空间有独立的连接、房间和事件
```

### 使用命名空间

```javascript
// 服务器端
const chatNsp = io.of('/chat')
const notifNsp = io.of('/notifications')

// 聊天命名空间
chatNsp.on('connection', (socket) => {
  console.log('聊天连接:', socket.id)
  
  socket.on('message', (data) => {
    chatNsp.emit('message', data)
  })
})

// 通知命名空间
notifNsp.on('connection', (socket) => {
  console.log('通知连接:', socket.id)
  
  socket.on('notify', (data) => {
    notifNsp.emit('notify', data)
  })
})
```

```javascript
// 客户端
// 连接聊天命名空间
const chatSocket = io('http://localhost:3001/chat')

chatSocket.on('connect', () => {
  console.log('聊天连接成功')

// 连接通知命名空间

  console.log('通知:', data)

```

## 实战：协同编辑（类似 Google Docs）

### 思路

```
1. 用户 A 编辑文档 → 发送变更到服务器
2. 服务器广播变更给所有在线用户
3. 用户 B 收到变更 → 应用到本地文档
```

### 简化实现

```javascript
// 服务器端
io.on('connection', (socket) => {
  // 用户加入文档
  socket.on('join-document', (docId) => {
    socket.join(docId)
  })
  
  // 接收文档变更
  socket.on('document-change', (data) => {
    const { docId, changes } = data
    
    // 广播给文档的其他编辑者（不包括发送者）
    socket.to(docId).emit('document-change', changes)
  })
})
```

```javascript
// 客户端
const socket = io('http://localhost:3001')

// 加入文档
socket.emit('join-document', 'doc-123')

// 监听文档变更
socket.on('document-change', (changes) => {
  // 应用变更到本地编辑器
  applyChangesToEditor(changes)
})

// 发送变更
function onEditorChange(changes) {
  socket.emit('document-change', {
    docId: 'doc-123',
    changes: changes
  })
}
```
